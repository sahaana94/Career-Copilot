# Referral Notes

A referral note is a short message the user sends (or hands to a contact to send on their behalf) asking for a referral or introduction at a company they want to work for. The goal is to make it as easy as possible for the contact to say yes and act — that means short, warm, specific, and low-effort to forward.

## 1. Gather context

Ask for whatever isn't already obvious from the conversation:
- **Who is this for?** Name, and the user's relationship to them (former colleague, college friend, LinkedIn connection, etc.) — this shapes the tone (a note to a close former teammate reads differently than one to a warm-but-distant connection).
- **Which role and company?** If a job description was discussed earlier in the conversation, use it.
- **What's the ask, specifically?** Usually one of: refer me through the formal referral system, forward my resume to the hiring manager/recruiter, or just an intro/conversation. If unclear, default to the gentler ask (intro or "happy to send my resume if useful") rather than assuming the most demanding one.

## 2. Build the "why I'm a fit" content

If a fit assessment was already produced for this role earlier in the conversation, reuse its **Top Strengths for This Role** — that's exactly the right content, already vetted for honesty. If not, pull 1–2 of the strongest, most relevant points from `Career Profile/profile.md`'s Work History / Core Strengths that map to this specific role.

Keep it to 1–2 points. A referral note isn't a cover letter — it should give the contact just enough to feel comfortable forwarding it, not a full case for hiring.

## 3. Output format

Default to a short note suitable for a DM, Slack message, or short email. Use this shape:

```markdown
# Referral Note — [Company]
**Role:** [Title]

Hi [Contact Name],

[1–2 sentences: warm, specific opener — reference the relationship/how you know them, and that you saw they're at [Company] / noticed the [Role] opening there.]

[1–2 sentences: brief, concrete pitch — who the user is professionally and the 1–2 strongest points of overlap with this role, drawn from the fit assessment or profile.]

[1 sentence: the ask — e.g., "Would you be open to referring me, or pointing me to the right person? Happy to send over my resume or anything else that'd help."]

Thanks so much,
[User's name]
```

If the user asks for a longer version (e.g., to send directly to a hiring manager rather than through a personal contact), expand the middle paragraph slightly with one more concrete accomplishment, but keep the whole thing under ~150 words — long referral asks get skimmed or ignored.

## 4. Tone

- Warm but not gushing. This is a real ask of a real relationship.
- Specific enough that the contact doesn't have to do research to understand why the user wants this role or why they'd be good at it.
- Make the ask easy to act on — name the concrete next step (refer, forward, intro) rather than leaving it vague.
- Don't oversell. The same honesty principle from fit scoring applies: a note that's accurate about the user's background builds more trust with the contact than one that stretches.

## 5. File handling

Save as `[Company]_Referral_Note.md` (or `.docx` if the user prefers, matching whatever format other documents in their company folder use). If a folder for this company/application already exists, save there.

## After drafting

If interview prep hasn't been built yet and the user mentions an interview is scheduled or likely, offer `references/interview-prep.md` as the natural next step.
